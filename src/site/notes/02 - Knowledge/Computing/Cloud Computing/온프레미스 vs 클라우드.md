# 온프레미스 vs 클라우드

> 기업이 IT 인프라를 어디에 두고 운영하느냐에 따른 두 가지 방식

## 온프레미스 (On-Premises)

- 기업이 **자체 데이터센터**에 서버·스토리지·네트워크를 직접 구축하고 운영
- 초기 구매·설치·유지보수 비용이 높음
- 보안과 데이터 제어에 유리 (외부 반출 없음)
- 규모 변경이 어렵고 느림 (하드웨어 추가 필요)

## 클라우드 (Cloud)

- AWS, GCP, Azure 같은 **외부 클라우드 공급자의 인프라**를 임대해 사용
- 사용한 만큼 지불하는 **종량제** 방식
- 필요에 따라 빠르게 확장/축소 가능
- 물리 서버 관리 부담 없음

## 비교

| 항목 | 온프레미스 | 클라우드 |
|------|-----------|----------|
| **초기 비용** | 높음 (장비 구매) | 낮음 (구독/종량제) |
| **운영 관리** | 직접 관리 | 공급자가 인프라 관리 |
| **보안/제어** | 완전한 통제 가능 | 공급자 정책에 의존 |
| **확장성** | 느리고 제한적 | 빠르고 유연함 |
| **장애 대응** | 자체 대응 | 공급자 SLA 활용 |

## 하이브리드 클라우드

- 온프레미스 + 클라우드를 혼합 운영
- 민감 데이터는 사내, 연산 집약 작업은 클라우드로 분리

## 면접 포인트

- **온프레미스 선택 기준**: 규제 산업(금융·의료), 민감 데이터, 예측 가능한 고정 부하
- **클라우드 선택 기준**: 스타트업, 부하 변동이 크거나 예측 불가능, 빠른 출시 필요
- **클라우드 단점**: 인터넷 의존성, 데이터 주권 이슈, 장기 운영 시 비용 증가 가능
