# LAN (Local Area Network)

> 토폴로지, IEEE 802 모델, MAC, 브리지, 스위치, VLAN, 이더넷, 무선 LAN

## 토폴로지 (Topology)

### 버스 토폴로지
- 모든 스테이션이 직선형 전송 매체(버스)에 탭으로 직접 연결
- 전송 신호는 양방향으로 전파 → 모든 스테이션이 수신 가능
- 버스 양 끝에 종단기(Terminator)로 신호 흡수

### 스타 토폴로지
- 각 스테이션이 중앙 노드(허브/스위치)에 연결
- 물리적으로는 스타, 논리적으로는 버스
- 중앙 노드가 프레임 스위치로 작동 가능

## IEEE 802 참조 모델

| 계층 | 역할 |
|------|------|
| **물리 계층** | 전송 매체/토폴로지, 신호 인코딩/디코딩, 비트 전송 |
| **LLC (논리 링크 제어)** | 상위 계층 인터페이스, 흐름 제어, 오류 제어 |
| **MAC (매체 접근 제어)** | 프레임 조립/분해, 주소 인식, 오류 검출, LAN 전송 매체 접근 제어 |

### LLC 서비스

| 서비스 | 설명 |
|--------|------|
| **확인되지 않은 연결 없는 서비스** | 데이터그램 스타일, 전달 보장 없음, 단순·빠름 |
| **연결 모드 서비스** | 논리적 연결 설정, 흐름/오류 제어 제공 |
| **확인된 연결 없는 서비스** | 데이터그램이 확인되지만 논리적 연결은 없음 |

### MAC 접근 방식
- **Synchronous**: 연결에 전용 용량 할당 (고정적 → 비효율)
- **Asynchronous**: 수요에 따라 유연하게 할당 (Round robin, Reservation, Contention)

## 네트워크 장비 (계층별)

### 1계층 - 리피터, 허브
- **리피터**: 신호 증폭 장치 (현재는 다른 장비에 통합)
- **허브**: 리피터 + 다중 장비 연결, CSMA/CD 방식, 충돌 도메인 공유

### 2계층 - 브리지, 스위치

#### 브리지 (Bridge)
- 동일한 물리/링크 계층 프로토콜의 유사한 LAN 연결
- 5가지 기능:
  - **Learning**: 소스 MAC 주소를 테이블에 저장
  - **Flooding**: 목적지 정보 없으면 모든 포트로 전송
  - **Forwarding**: 목적지 포트 알면 해당 포트로만 전송
  - **Filtering**: 같은 세그먼트 내 통신은 브리지를 건너지 못하게 차단
  - **Aging**: 300초 동안 해당 MAC 수신 없으면 테이블에서 삭제
- 프레임 처리: **소프트웨어**, Store-and-Forward 방식

#### 스위치 (Switch)
- 브리지의 상위 호환 (처리 방식이 **하드웨어**)
- Store-and-Forward: 프레임 전체 수신 후 전달 (신뢰성 높음)
- Cut-through: 목적지 주소 인식 즉시 전달 (빠르지만 오류 프레임 전파 가능)
- L2/L3/L4/L7 스위치로 계층별 분류

#### 스패닝 트리 알고리즘
- 브리지가 자동으로 라우팅 테이블 개발
- 닫힌 루프 없이 연결 유지
- 세 메커니즘: 프레임 포워딩, 주소 학습, 루프 해결

### 3계층 - 라우터
- 네트워크 간 최적 경로 설정 및 데이터 전송
- 소프트웨어 방식 → L3 스위치보다 느리나 다양한 기능

## VLAN (Virtual LAN)
- 물리적 위치에 제한되지 않고 공통 LAN처럼 통신하는 스테이션 그룹
- 브로드캐스트 도메인 분리
- 멤버십 기준: 포트 그룹, MAC 주소, 프로토콜 정보

## 이더넷 (Ethernet)

### CSMA/CD
- ALOHA → 슬롯 ALOHA → CSMA → CSMA/CD 발전
- 비영속적/1-영속적/p-영속적 CSMA
- **IEEE 802.3**: 1-영속적 알고리즘 + 이진 지수 백오프
  - 충돌 감지 시 재밍 신호 전송 → 랜덤 백오프 후 재전송
  - 16번 시도 실패 시 포기 및 오류 보고

## 무선 LAN (IEEE 802.11)

### MAC 계층 기능
- 신뢰할 수 있는 데이터 전달
- 접근 제어
- 보안

### DCF (분산 조정 기능)
- CSMA 알고리즘 사용 (무선에서는 충돌 감지 불가)
- IFS(Interframe Space)로 우선순위 부여:
  - **SIFS** (가장 짧음): ACK, CTS, Poll 응답에 사용 → 최고 우선순위
  - **PIFS**: 중앙 집중식 컨트롤러 폴에 사용
  - **DIFS** (가장 긺): 비동기 프레임 경합 시 사용

### 네 프레임 교환
1. 소스 → RTS (전송 요청)
2. 목적지 → CTS (전송 승인)
3. 소스 → Data
4. 목적지 → ACK

### IEEE 802.11 버전
- **802.11n**: MIMO 안테나, 용량 증가
- **802.11ac**: MU-MIMO, A-MPDU 집합
- **802.11ad**: 60GHz 대역, 거대한 채널 대역폭 (2160MHz)
